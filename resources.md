# Resources & Tools

A curated collection of learning platforms, communities, tools, and reading for urban data professionals. Organised by category to help you find what you need.

---

## Learning Platforms

### Structured Courses
- **[The Carpentries](https://carpentries.org/)** — Data Carpentry and Software Carpentry workshops. Excellent hands-on introductions to data skills for researchers and professionals. Often run at Australian universities.
- **[Coursera](https://www.coursera.org/)** — University-backed courses including *Python for Everybody* (Michigan), *GIS Specialisation* (UC Davis), Andrew Ng's *Machine Learning*
- **[edX](https://www.edx.org/)** — Free courses from MIT, Harvard, and others. Look for data science and spatial analysis offerings.
- **[DataCamp](https://www.datacamp.com/)** — Interactive Python, R, and SQL courses with a data focus
- **[Codecademy](https://www.codecademy.com/)** — Interactive coding courses for beginners
- **[ESRI Academy](https://www.esri.com/training/)** — GIS-specific training from Esri (free and paid)
- **[QGIS Documentation & Tutorials](https://qgis.org/en/docs/)** — Official learning resources for QGIS

### Urban Data Specific
- **[School of Cities — Urban Data Storytelling](https://schoolofcities.github.io/urban-data-storytelling/)** — Open course from the University of Toronto on telling stories with urban data
- **[PIA PlanTech Resources](https://www.planning.org.au/pia/policy-advocacy/plantech-resources.aspx)** — Planning Institute of Australia's digital planning resources
- **[Digital Planning Core Competencies](https://www.planning.org.au/pia/policy-advocacy/plantech.aspx?hkey=360d84d4-a1ef-4f51-95ca-44aaba914de2)** — PIA's framework for digital skills in planning

## Essential Tools

### Text Editors (Your Best Friend)
A good text editor is essential for working with data files, code, and configuration. These handle CSV, JSON, XML, and code files far better than Word or Notepad.

- **[Notepad++](https://notepad-plus-plus.org/)** — Windows, free, lightweight, excellent for data files
- **[Notepad Next](https://github.com/dail8859/NotepadNext)** — Cross-platform alternative inspired by Notepad++
- **[VS Code](https://code.visualstudio.com/)** — Microsoft's free code editor with excellent extension ecosystem

### GIS Software
- **[QGIS](https://qgis.org/)** — Free, open-source GIS. Start here.
- **[ArcGIS Pro](https://www.esri.com/en-us/arcgis/products/arcgis-pro/overview)** — Industry standard (paid, but available through many organisations)
- **[Google Earth Pro](https://www.google.com/earth/versions/)** — Free, useful for quick spatial visualisation

### Programming Environments
- **[Python](https://www.python.org/)** — via [Anaconda](https://www.anaconda.com/) (includes Jupyter, Pandas, etc.) or [uv](https://github.com/astral-sh/uv) for lightweight package management
- **[R](https://www.r-project.org/)** — via [RStudio](https://posit.co/download/rstudio-desktop/)
- **[Jupyter Notebooks](https://jupyter.org/)** — Interactive coding environment (included in Anaconda)

### Data Visualisation
- **[Tableau Public](https://public.tableau.com/)** — Free interactive dashboards
- **[Power BI](https://powerbi.microsoft.com/)** — Microsoft's analytics platform (free desktop version)
- **[Streamlit](https://streamlit.io/)** — Python library for building data apps quickly
- **[Plotly / Dash](https://plotly.com/)** — Interactive charts and dashboards in Python

### Database Tools
- **[PostgreSQL](https://www.postgresql.org/)** + **[PostGIS](https://postgis.net/)** — Spatial database. The gold standard for spatial data management.
- **[DBeaver](https://dbeaver.io/)** — Free database client for PostgreSQL, SQLite, and others
- **[SQLite](https://www.sqlite.org/)** — Lightweight file-based database (great for learning)

### Data Conversion & Cleaning
- Online format converters for common data types
- **[OpenRefine](https://openrefine.org/)** — Powerful free tool for cleaning messy data
- **[GDAL/OGR](https://gdal.org/)** — Command-line tools for geospatial format conversion

### AI & LLM Tools
AI assistants can accelerate learning and help with code writing, data exploration, and research. Use them as a complement to building your own understanding.

- Useful for: explaining concepts, debugging code, exploring data structures, generating initial scripts
- Limitations: agents can produce incorrect code or analysis; always verify outputs
- Best practice: use AI to learn *how* to do things, then build the skill yourself

## Data Principles & Standards

### Key Frameworks
- **[FAIR Principles](https://www.go-fair.org/fair-principles/)** — Findable, Accessible, Interoperable, Reusable
- **[CARE Principles](https://www.gida-global.org/care)** — Collective benefit, Authority to control, Responsibility, Ethics (for Indigenous data governance)
- **[Open Data Charter](https://opendatacharter.net/)** — International principles for open data

### Australian Standards
- **[ANZLIC Metadata Profile](https://www.anzlic.gov.au/)** — Australian and New Zealand spatial metadata standard
- **[AS/NZS ISO 19115](https://www.iso.org/standard/53798.html)** — Geographic information metadata standard
